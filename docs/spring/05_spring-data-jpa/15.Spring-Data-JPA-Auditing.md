# Spring Data JPA Auditing

## Purpose

`BaseAuditableEntity` is a reusable base class that automatically tracks:

* When a record was created
* When a record was last modified
* Who created the record
* Who last modified the record

This avoids duplicating auditing fields across all entities.

---

## Typical Implementation

```java
@MappedSuperclass
@EntityListeners(AuditingEntityListener.class)
public abstract class BaseAuditableEntity {

    @CreatedDate
    @Column(name = "created_date", nullable = false, updatable = false)
    private LocalDateTime createdDate;

    @LastModifiedDate
    @Column(name = "last_modified_date")
    private LocalDateTime lastModifiedDate;

    @CreatedBy
    @Column(name = "created_by", updatable = false)
    private String createdBy;

    @LastModifiedBy
    @Column(name = "last_modified_by")
    private String lastModifiedBy;

    @PrePersist
    protected void onCreate() {
        LocalDateTime now = LocalDateTime.now();
        if (createdDate == null) {
            createdDate = now;
        }
        if (lastModifiedDate == null) {
            lastModifiedDate = now;
        }
    }

    @PreUpdate
    protected void onUpdate() {
        lastModifiedDate = LocalDateTime.now();
    }

}
```

# Annotation Details

## 1. @MappedSuperclass

### Purpose

Marks the class as a JPA mapping superclass.

The class itself:

* Is NOT an entity
* Does NOT have its own table
* Cannot be queried directly

Its fields and mappings are inherited by child entities.

### Example

```java
@Entity
public class Employee extends BaseAuditableEntity {

    private String name;
}
```

Hibernate treats it as:

```java
@Entity
public class Employee {

    private LocalDateTime createdDate;
    private LocalDateTime lastModifiedDate;
    private String createdBy;
    private String lastModifiedBy;

    private String name;
}
```

### Database Table

```sql
EMPLOYEE
---------
ID
NAME
CREATED_DATE
LAST_MODIFIED_DATE
CREATED_BY
LAST_MODIFIED_BY
```

No table is created for `BaseAuditableEntity`.

### If @MappedSuperclass Is Missing

```java
@EntityListeners(AuditingEntityListener.class)
public abstract class BaseAuditableEntity {
    
}
```

JPA may ignore the inherited mappings.

Common symptoms:

* Columns not generated
* Auditing fields not persisted
* Parent mappings ignored
* Auditing appears broken

For reusable base classes containing JPA annotations, `@MappedSuperclass` is typically required.

---

## 2. @EntityListeners(AuditingEntityListener.class)

### Purpose

Registers Spring's auditing listener.

The listener reacts to entity lifecycle events:

```text
INSERT -> PrePersist
UPDATE -> PreUpdate
```

and automatically populates auditing fields.

### Without It

Fields marked with:

```
@CreatedDate
@LastModifiedDate
@CreatedBy
@LastModifiedBy
```

will not be populated automatically.

Values will remain `null` unless manually assigned.

---

## 3. @CreatedDate

```java
@CreatedDate
private LocalDateTime createdDate;
```

### Purpose

Automatically stores the timestamp when the entity is first inserted.

### Example

```
employeeRepository.save(employee);
```

Before save:

```text
createdDate = null
```

After save:

```text
createdDate = 2026-06-15T10:00:00Z
```

### Typical Column Configuration

```
@Column(nullable = false, updatable = false)
```

Meaning:

* Cannot be null
* Never updated after insert

---

## 4. @LastModifiedDate

```java
@LastModifiedDate
private LocalDateTime lastModifiedDate;
```

### Purpose

Automatically updated whenever the entity changes.

### Example

Initial Insert:

```text
createdDate      = 10:00
lastModifiedDate = 10:00
```

After Update:

```text
createdDate      = 10:00
lastModifiedDate = 11:15
```

Only the modification timestamp changes.

---

## 5. @CreatedBy

```java
@CreatedBy
private String createdBy;
```

### Purpose

Stores the user who created the record.

Spring retrieves this value from an `AuditorAware<T>` implementation.

Example:

```java
@Bean
AuditorAware<String> auditorProvider() {
    return () -> Optional.of(
        SecurityContextHolder
            .getContext()
            .getAuthentication()
            .getName()
    );
}
```

Result:

```text
createdBy = mohan.medarametla
```

---

## 6. @LastModifiedBy

```java
@LastModifiedBy
private String lastModifiedBy;
```

### Purpose

Stores the user who most recently modified the record.

Example:

```text
Created By      : Mohan
Last Modified By: John
```

# Column Configuration

## updatable = false

```
@Column(updatable = false)
```

Typically used on:

```
createdDate
createdBy
```

Meaning:

```text
INSERT -> allowed
UPDATE -> ignored
```

These values never change after creation.

---

## nullable = false

```
@Column(nullable = false)
```

Creates:

```sql
created_date TIMESTAMP NOT NULL
```

Prevents storing null values.

# How Auditing Works Internally

## Insert Flow

Application:

```
employeeRepository.save(employee);
```

Spring:

```text
AuditingEntityListener
        ↓
@PrePersist
        ↓
Set CreatedDate
Set LastModifiedDate
Set CreatedBy
Set LastModifiedBy
        ↓
INSERT INTO DATABASE
```

Result:

```text
createdDate      = current timestamp
lastModifiedDate = current timestamp
createdBy        = current user
lastModifiedBy   = current user
```

---

## Update Flow

Application:

```
employee.setName("Updated Name");
employeeRepository.save(employee);
```

Spring:

```text
AuditingEntityListener
        ↓
@PreUpdate
        ↓
Update LastModifiedDate
Update LastModifiedBy
        ↓
UPDATE DATABASE
```

Result:

```text
createdDate      = unchanged
createdBy        = unchanged

lastModifiedDate = updated
lastModifiedBy   = updated
```

# Required Configuration

## Enable Auditing

```java
@Configuration
@EnableJpaAuditing
public class JpaConfig {
}
```

Required for Spring Data JPA auditing.

---

## Provide AuditorAware

Required only if using:

```
@CreatedBy
@LastModifiedBy
```

Example:

```java
@Bean
AuditorAware<String> auditorProvider() {
    return () -> Optional.of("system-user");
}
```

# Common Real-World Pattern

```java
@MappedSuperclass
@EntityListeners(AuditingEntityListener.class)
@Getter
@Setter
public abstract class BaseAuditableEntity {

    @CreatedDate
    @Column(nullable = false, updatable = false)
    private LocalDateTime createdDate;

    @LastModifiedDate
    private LocalDateTime lastModifiedDate;

    @CreatedBy
    @Column(updatable = false)
    private String createdBy;

    @LastModifiedBy
    private String lastModifiedBy;
}
```

Usage:

```java
@Entity
public class Customer extends BaseAuditableEntity {
}

@Entity
public class Order extends BaseAuditableEntity {
}

@Entity
public class Invoice extends BaseAuditableEntity {
}
```

All entities automatically receive auditing support without duplicating code.

# Quick Interview Answer

**Why use @MappedSuperclass?**

To share common JPA mappings (id, auditing fields, timestamps, etc.) across multiple entities without creating a separate database table.

**Why use @EntityListeners(AuditingEntityListener.class)?**

To allow Spring Data JPA to automatically populate auditing fields during insert and update operations.

**What happens if @MappedSuperclass is removed?**

Inherited JPA mappings may not be recognized, auditing columns may not be created, and auditing functionality can stop working.

**What happens if @EntityListeners is removed?**

Auditing annotations such as `@CreatedDate` and `@LastModifiedDate` will not be populated automatically.

**What happens if @EnableJpaAuditing is missing?**

Spring auditing infrastructure is not activated, so auditing fields will not be managed automatically.
